# Resolve Git Pull Conflict - package-lock.json

## Problem
When pulling from SIT branch, you get:
```
error: Your local changes to the following files would be overwritten by merge:
        frontend/package-lock.json
```

## Solution Options

### Option 1: Stash Local Changes (Recommended)
This saves your local changes and applies them after pull:

```bash
cd /opt/tas-production

# Stash local changes
git stash

# Pull the updates
git pull origin SIT

# If you need your stashed changes back (usually not needed for package-lock.json)
# git stash pop
```

### Option 2: Discard Local Changes (If not important)
If the local package-lock.json changes are not important:

```bash
cd /opt/tas-production

# Discard local changes to package-lock.json
git checkout -- frontend/package-lock.json

# Pull the updates
git pull origin SIT
```

### Option 3: Commit Local Changes First
If you want to keep your local changes:

```bash
cd /opt/tas-production

# Commit local changes
git add frontend/package-lock.json
git commit -m "Update package-lock.json"

# Pull (may create merge conflict)
git pull origin SIT

# If there's a merge conflict, resolve it and commit
# git add frontend/package-lock.json
# git commit -m "Merge package-lock.json"
```

## Recommended Approach

For `package-lock.json`, **Option 2 (Discard)** is usually best because:
- package-lock.json is auto-generated
- The remote version is the source of truth
- You can regenerate it with `npm install` if needed

## After Pulling

If you need to regenerate package-lock.json:

```bash
cd /opt/tas-production/frontend
npm install
```

This will update package-lock.json to match package.json.
